textup（文字起こし・要約）取扱説明書

会議や講義の音声を録音すると、AIが自動で文字起こしし、さらに要約まで作成してくれるアプリです。

このアプリでできること

  • マイクで会議・講義などの音声を録音します。
  • 録音した音声をAIが文字起こし（テキスト化）します。
  • 文字起こし結果から、AIが要約（議事録風のまとめ）を作成します。
  • 話者ごとの編集（話者名の修正など）ができます。
  • 文字起こしに失敗した場合に再実行できます。
  • セッション（録音1回分）ごとに結果を保存し、後から見返せます。
  • スマートフォンでもアプリのように使える（PWA・ホーム画面に追加可能）。

※ 文字起こしの精度や対応言語、対応する録音時間の上限などの詳細は未確認です。重要な記録は元音声や人の確認も併用してください。

想定ユーザー・利用シーン

  • 会議の議事録作成を効率化したいビジネス利用者。
  • 講義・セミナーの内容を記録・要約して復習したい人。
  • 取材・打ち合わせのメモ取りを音声から自動化したい人。

動作環境・前提

  • マイクが使えるパソコン・スマートフォンのWebブラウザ（最新版の Chrome などを推奨）。
  • マイクの使用許可が必要です。
  • インターネット接続。
  • ログイン（サインイン）が必要です。
  • AIによる文字起こし・要約に Google の Gemini API を利用します。

セットアップ

通常の利用者は、公開URLにアクセスしてログインするだけで利用できます。スマートフォンでは、ブラウザの「ホーム画面に追加」でアプリのように使えます。

開発者がローカルで動かす場合は「開発者向けメモ」を参照してください。

使い方（ステップbyステップ）

  1. アプリのURLを開き、サインインします。
  2. ダッシュボードから新しい録音（セッション）を開始します。
  3. 初回はブラウザからマイクの使用許可を求められるので「許可」します。
  4. 録音設定（必要に応じて）を確認し、録音を開始します。音声レベルメーターで入力を確認できます。
  5. 会議・講義が終わったら録音を停止します。音声が文字起こし処理に送られます。
  6. しばらく待つと文字起こし結果（発言一覧）が表示されます。状態は自動で更新されます。
  7. 必要なら話者名を編集したり、失敗した部分を再実行します。
  8. 「要約を作成」を実行すると、AIが内容をまとめた要約を生成します。
  9. 結果はセッションとして保存され、あとから一覧で見返せます。

主な機能

  • 録音: ブラウザのマイクで録音し、WAV形式にエンコード。音声レベルメーター付き。
  • 文字起こし: 録音音声をAIでテキスト化。失敗時の再実行に対応。
  • 要約生成: 文字起こし結果からAIが要約（Markdown表示）。
  • 話者編集: 話者名などを編集するダイアログ。
  • セッション管理: 録音ごとに結果を保存し、状態を自動更新（ポーリング）。
  • PWA対応: ホーム画面追加・サービスワーカー登録に対応。

よくある質問・トラブルシューティング

  • Q. 録音できません。 → ブラウザのマイク使用許可を確認してください。許可をブロックしている場合は設定から解除します。
  • Q. 文字起こしが終わりません。 → 処理に時間がかかる場合があります。状態表示が更新されるのを待ち、失敗時は再実行ボタンを使ってください。
  • Q. 要約がうまく出ません。 → 文字起こし結果が短い・不正確だと要約も影響を受けます。録音環境（雑音）を見直してください。
  • Q. AIの無料枠でエラーになります。 → 利用モデルは自動でフォールバックする設計です。時間をおいて再実行してください。
  • Q. スマホでアプリのように使いたい。 → ブラウザの「ホーム画面に追加」を利用してください（PWA対応）。
